# Purpose

Use this skill when the app has one main page with tabs, segmented controls, panels, or view switches that need to work well on phones.

# Instructions

1. Make the primary navigation obvious and reachable with one hand.
2. Keep the number of top-level actions limited on narrow screens.
3. Ensure tab labels remain readable without wrapping awkwardly.
4. Prefer sticky or bottom-reachable navigation when it fits the existing app.
5. Preserve back-navigation expectations where possible.
6. Prevent content jumps when switching views.
7. Make each view resilient to tall content and virtual keyboard changes.

# Output expectations

- State whether current tab structure is mobile-friendly.
- Note any navigation change that could affect user habits.
